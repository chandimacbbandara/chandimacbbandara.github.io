Student Concern Management System (SCMS) – Web

Centralized web system for student concerns with AI-assisted insights.

Designed and built the web portal to collect, triage, and track student concerns across departments, enabling faster responses and better oversight.

  • Role-based dashboards for students, staff, and admins.
  • AI-assisted concern summaries and categorization.
  • Resolution analytics for continuous improvement.
Java Spring Boot Thymeleaf SQL Server Gemini API
GitHub

Overview

SCMS is a full-stack web system for managing student concerns from submission to resolution. It supports Students, Admins, and Owner users with AI-assisted moderation and priority prediction.

Highlights

  • Role-based login with complete concern lifecycle management.
  • Evidence uploads and linked concerns for threaded history.
  • Community forum with moderation and rules acceptance.
  • AI moderation and priority prediction in submit/update flows.
  • Dashboards for admins and owners with analytics and reports.
  • Notification center and feedback modules for closure quality.
  • Notification center and feedback modules for closure quality.

Roles

  • Students: submit concerns, chat, schedule meetings, and give feedback.
  • Admins: review, prioritize, respond, and moderate community content.
  • Owner: manage admins, analytics, FAQs, tips, and notifications.

Tech Stack

  • Java 21, Spring Boot 4.0.2, Spring MVC + Thymeleaf.
  • Spring Data JPA (Hibernate), MySQL Connector/J.
  • Spring Mail (SMTP), Spring Security Crypto (BCrypt).
  • TensorFlow Java dependency and Python AI stack (torch, transformers, scikit-learn).
  • Maven Wrapper (mvnw).
